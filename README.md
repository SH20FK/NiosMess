# NiosMess — документация по проекту

## Описание
NiosMess — десктопный клиент мессенджера на Python (PyQt5). Клиент подключается к серверу по HTTP, поддерживает авторизацию, список чатов, обмен текстовыми сообщениями и файлами, а также просмотр профиля пользователя.

## Основные возможности
- Авторизация и регистрация через сервер (с поддержкой подтверждения по коду).
- Автоматический вход по сохранённой сессии.
- Список чатов с индикацией онлайн-статуса.
- Обмен сообщениями в реальном времени (периодическая синхронизация).
- Отправка файлов и скачивание вложений.
- Просмотр профиля выбранного пользователя.
- Автозапуск (в режиме exe) и скрытие консоли на Windows.

## Требования
- Python 3.8+ (рекомендуется).
- Windows (на Linux/macOS часть функций автозапуска не применяется).
- Пакеты:
  - PyQt5
  - requests

Установка зависимостей:
```bash
pip install PyQt5 requests
```

## Быстрый старт
Запуск клиентского приложения:
```bash
python starter.py
```

Альтернативно, можно запустить окно авторизации напрямую:
```bash
python login.py
```

> В `starter.py` предусмотрен флаг `--nogui`, который запускает приложение без отображения окна, но QApplication всё равно создаётся.

## Конфигурация и пути
Основные константы определены в `handlers.py`:
- `SERVER_URL`: адрес сервера, по умолчанию `http://144.31.1.79:27580`.
- `SESSION_FILE`: файл сессии пользователя (в домашней директории).
- `DOWNLOAD_DIR`: папка для скачанных файлов (`~/Downloads/NiosMess`).

Если требуется сменить сервер, отредактируйте `SERVER_URL` в `handlers.py`.

## Архитектура проекта
- `starter.py` — точка входа, скрывает консоль, добавляет автозапуск (если exe), запускает GUI.
- `login.py` — окно авторизации/регистрации, хранит сессию и передаёт управление мессенджеру.
- `messenger.py` — основная логика чатов, синхронизация, отправка сообщений/файлов.
- `handlers.py` — общий UI/worker-код: виджеты, стили, загрузка файлов, работа с токеном.
- `obresfucate.py` — "обфускация" текста для отправки (это не шифрование).

## Сетевые эндпоинты (клиент ожидает на сервере)
Клиент обращается к следующим маршрутам:
- `POST /login` — вход по email/паролю.
- `POST /register` — регистрация нового пользователя.
- `POST /check_session` — проверка сохранённой сессии.
- `POST /ping` — поддержание онлайн-статуса.
- `GET /get_chats` — список чатов пользователя.
- `GET /get_messages` — история сообщений между двумя пользователями.
- `POST /send_message` — отправка сообщения.
- `POST /upload` — загрузка файла.
- `GET /download/{filename}` — скачивание файла.
- `GET /search_users` — поиск пользователей.
- `GET /get_user_info` — профиль пользователя.

## Формат сообщений и файлов
- Сообщения отправляются текстом через `send_message`.
- Файлы отправляются через `upload` и отображаются как специальные записи `FILE: имя_файла`.
- Картинки/аудио распознаются по расширению и отображаются в отдельном UI-блоке.

## Сессии
Сессия хранится локально в `niosmess_session.json` в домашней директории пользователя. Файл содержит `token` и `username` (и другие поля, которые возвращает сервер).

## Обфускация сообщений
Модуль `obresfucate.py` выполняет простую подстановку символов (map-замена). Это **не является криптографическим шифрованием** и не гарантирует безопасность.

## Автозапуск (Windows)
`starter.py` при запуске exe (PyInstaller) добавляет приложение в автозагрузку пользователя через ключ реестра:
```
HKCU\Software\Microsoft\Windows\CurrentVersion\Run
```
В режиме `.py` автозапуск не добавляется.

## Диагностика и типовые проблемы
- Если русские сообщения отображаются как набор символов, убедитесь, что файлы сохранены в UTF-8.
- При отсутствии сети клиент может показывать ошибки соединения или не обновлять список чатов.
- Если сервер недоступен, проверьте `SERVER_URL` и доступность эндпоинтов.
