1) Каталог недостающих фич.
2) Максимально подробный разбор каждой фичи с точки зрения фронта и бэка.

## 2) Подробный разбор каждой фичи (Front / Back)

Ниже по каждой функции: что должно быть на фронте и что обязательно нужно на бэке.

---

### M01. Реалтайм-обновления (WebSocket/WS)
**Фронт:**
- Подключение WS после логина.
- События: new_message, edit_message, delete_message, typing, presence, read_receipt.
- Обновление UI без перезагрузки (append в список).
- Реконнект при обрыве.

**Бэк:**
- WS шлюз + авторизация по токену.
- Роутинг событий по chat_id.
- Очередь событий для оффлайн пользователей.

---

### M02. Статусы доставки и прочтения
**Фронт:**
- Отображение статусов (✓, ✓✓, «прочитано»).
- Анимация подтверждения.

**Бэк:**
- Поля message_status, read_at.
- Эндпоинт /mark_read (если WS недоступен).
- Рассылка read_receipt по участникам.

---

### M03. Серверные unread-счётчики
**Фронт:**
- Отображение server unread_count в списке.
- Обнуление при открытии чата.

**Бэк:**
- Хранить unread per user/per chat.
- /collective/mark_read, /read_message.

---

### M04. Мульти-девайс синхронизация

- Список устройств.
